| Ref | 항목 | 체크 |

| NFR-04 | Domain 경로에서 호출자 `grid` **비변형** | [ ] |

### 1.2 Entity (`src/magicsquare/entity`) — 필요 시

- [ ] 격자·좌표 등 Domain이 쓰는 **순수 데이터/타입** 정리 (과도한 로직은 Domain에 둠)

### 1.3 Boundary (`src/magicsquare/boundary`) — UX/API Contract

| Ref | 항목 | 체크 |
|-----|------|------|
| FR-01 | 4x4 크기 검증 → `INVALID_SIZE` (§5 문구 정확 일치) | [ ] |
| FR-01 | 셀 값 `0` 또는 `1..16` → `INVALID_VALUE_RANGE` | [ ] |
| FR-01 | `0` 정확히 2개 → 아니면 `INVALID_ZERO_COUNT` | [ ] |
| FR-01 | `0` 제외 `1..16` 중복 없음 → `DUPLICATE_VALUES` | [ ] |
| §5 | 위반 시 **Domain 미호출** 구조로 연결 | [ ] |
| §5 | 검증 **우선순위**: SIZE → RANGE → ZERO_COUNT → DUPLICATE | [ ] |

### 1.4 Control (`src/magicsquare/control`)

- [ ] Boundary 통과 시 Domain 호출, 성공 시 **`int[6]`** (`[r1,c1,n1,r2,c2,n2]` **1-index**, PRD §6.2)
- [ ] Domain “해 없음” → `{"code":"NO_SOLUTION","message":...}` (§5 정확 일치)
- [ ] Boundary 오류는 **dict** 로만 반환, 성공은 **list** 길이 6
