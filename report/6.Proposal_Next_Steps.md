# MagicSquare 4x4 제안사항

현재 구현 상태(입력 계약/도메인 분리, 22개 테스트 통과)를 기준으로, 다음 개선을 제안합니다.

## 1) PRD Traceability 명시 강화

- 테스트 파일 상단 또는 케이스명에 `FR`, `AC`, `TC-ID`를 직접 매핑합니다.
- 예: `test_e2e_ok_01_returns_int6_for_valid_solvable_grid`에 `E2E-OK-01`, `FR-05` 주석 추가.
- 효과: 리뷰/회귀 시 요구사항 누락 여부를 빠르게 확인할 수 있습니다.

## 2) Boundary/Domain 커버리지 게이트 자동화

- `pytest-cov`를 `dev` 의존성에 포함하고, CI에서 다음 기준을 강제합니다.
- Domain >= 95%
- Boundary >= 85%
- 효과: 리팩터링 시 NFR-01/02 저하를 조기에 차단할 수 있습니다.

## 3) 입력 불변성(NFR-04) 보호를 위한 회귀 테스트 확장

- 현재 E2E 1건의 비변형 테스트를 경계 케이스(오류 입력 포함)로 확장합니다.
- 예: 오류 반환 시에도 원본 `grid`가 변경되지 않는지 검증.
- 효과: Control/Domain 변경 중 부수효과 회귀를 방지합니다.

## 4) Domain 규칙 테스트 미세 강화

- `MagicSquareValidator`의 조기 실패(0 포함) 케이스를 명시적으로 테스트합니다.
- 행/열/대각 각각 단독 실패 케이스를 분리해 진단성을 높입니다.
- 효과: 실패 원인 파악이 쉬워지고 리팩터링 안전성이 올라갑니다.

## 5) 공개 API 계약 문서화

- `solve_magic_square(grid)`의 입력/출력 스키마를 `docs`에 별도 명세로 추가합니다.
- 성공: `int[6]` (`1-index` 좌표)
- 실패: `{code, message}` (`INVALID_*`, `NO_SOLUTION`)
- 효과: 호출자(테스트/CLI/향후 UI) 관점의 계약 일관성을 유지할 수 있습니다.
