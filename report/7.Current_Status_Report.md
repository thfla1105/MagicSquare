# MagicSquare 4x4 현재 진행 보고서

작성일: 2026-04-27

## 1. 작업 목적

본 프로젝트는 부분적으로 비어 있는 4x4 Magic Square를 입력받아:

- 유효한 입력이면 두 빈칸을 채우는 해를 반환하고
- 계약 위반 또는 해가 없으면 표준 `code`/`message`를 반환하는
- Python 기반 Dual-Track TDD 연습용 구조를 마련하는 것을 목표로 진행되었습니다.

## 2. 구현 현황 요약

현재 저장소에는 아래 항목이 구현되어 있습니다.

### 아키텍처/패키지

- `src/magicsquare/boundary`
  - 입력 계약 검증기(`InputValidator`)
  - 표준 오류 응답 계약(`contracts.py`)
- `src/magicsquare/domain`
  - 빈칸 탐색(`EmptyCellLocator`)
  - 누락 수 계산(`MissingNumbersResolver`)
  - 마방진 판정(`MagicSquareValidator`)
  - 두 번의 배치 시도 기반 해 결정(`MagicSquareResolver`)
- `src/magicsquare/control`
  - 유스케이스 진입점(`SolveMagicSquareUseCase`)
- `src/magicsquare/entity`
  - 입력 비변형 보조(`Grid`, 방어적 복사)
- 공개 API
  - `magicsquare.solve_magic_square(grid)`

### 계약 메시지/코드 반영

아래 오류 계약이 고정 문자열로 반영되어 있습니다.

- `INVALID_SIZE` / `Grid must be 4x4.`
- `INVALID_VALUE_RANGE` / `Each cell must be 0 or 1..16.`
- `INVALID_ZERO_COUNT` / `There must be exactly two zeros (empty cells).`
- `DUPLICATE_VALUES` / `Values 1..16 must not duplicate (excluding zeros).`
- `NO_SOLUTION` / `No placement makes a 4x4 magic square with magic sum 34.`

검증 우선순위도 다음 순서로 적용됩니다.

1. `INVALID_SIZE`
2. `INVALID_VALUE_RANGE`
3. `INVALID_ZERO_COUNT`
4. `DUPLICATE_VALUES`
